# 製品要求仕様

## 文書目的

この文書は、KotobaLink の現在の製品位置づけ、ユーザーロール、製品目標、MVP 範囲を整理するものです。

## 1. 製品位置づけ

KotobaLink は日本語学習サービス向けの学習支援プラットフォームとして位置づけられています。

現在の demo が表現している範囲:
- コース閲覧と購入
- 時間割確認
- lesson 内容確認
- assessment 確認
- 毎日学習タスク
- 宿題提出
- 通知
- イベント申込
- 学習成長表示

## 2. 対象ユーザー

### 受講生

現在の demo が直接対象としているロールです。

主な要求:
- 現在の学習状態を把握する
- コースと時間割を確認する
- 日常タスクを完了する
- 宿題を提出し、フィードバックを確認する
- 活動、コース、タスク通知を受け取る
- 成長推移を確認する

### 運営 / 教務 / 管理者

今後の管理画面で必要となるロールです。
- コース管理
- 排課管理
- イベント管理
- 通知管理
- 注文管理
- 会員管理
- 受講生記録管理

### 教師

完全版で必要となるロールです。
- lesson 内容管理
- 宿題講評
- 成長フィードバック
- assessment 結果管理

## 3. 現在の製品目標

1. 学習サービスのクローズドループをフロントエンド prototype として表現する
2. 将来の多端開発向けに構造を先に安定させる
3. コア業務フローを整理して実開発の準備を行う
4. 受講生側体験を起点に管理画面とバックエンド設計へ接続する

## 4. 現在の MVP 範囲

### 現在含まれる範囲

- ホーム概要
- コース一覧
- コース詳細
- 購入結果ページ
- 申込済みコース詳細
- lesson 詳細
- assessment 詳細
- 週時間割
- 毎日タスク
- 宿題ページ
- メッセージセンター
- マイページ
- 会員センター表示
- イベントページ

### 現在未実装だが今後必要な範囲

- ログイン認証
- 実注文と実支払い
- 実通知配信
- lesson / assessment の管理画面保守
- 宿題記録と教師講評
- 管理画面
