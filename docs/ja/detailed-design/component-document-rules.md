# 記述ルール

## 文書単位

控件文書は 1 控件 1 文書を原則とします。

控件に関する以下の内容は、分割せず同一文書にまとめます。
- 表示項目
- 参数
- 字段
- ボタン
- イベント
- 状態
- データ源
- 連動先
- 異常系
- 実 API 置換点

ページ文書は、控件の並び順とページ全体の導線のみを保持します。

## 必須記述項目

各控件文書には以下の項目を含めます。

1. 基本情報
2. 配置先
3. 責務範囲
4. 構造
5. 表示フィールド表
6. 入力パラメータ表
7. 操作項目表
8. 出力イベント表
9. 状態連動表
10. データ流入元 / 流出先
11. 例外条件表
12. 実 API 置換時の変更点

## ID ルール

- ページ ID: `PAGE-*`
- 共有控件 ID: `CMP-SHARED-*`
- 首页控件 ID: `CMP-HOME-*`

## フィールド表記ルール

フィールドは表形式で記述します。

- 表示名
- キー
- 型
- 必須 여부
- 例
- データ元
- 空値時挙動
- 連動先

## 操作記述ルール

各ボタン、各押下可能領域について以下を表形式で必ず記述します。

- 表示文言
- イベント名
- 発火条件
- 無効条件
- 押下後の処理
- 更新される状態
- 遷移先

## 現在の前提

現在の miniapp では以下の層が存在します。

- `mock/`
- `store/`
- `services/`
- `pages/`

控件文書では、データ源の記述をこの層に必ず接続します。
