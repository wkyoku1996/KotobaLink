# 产品需求说明

## 文档目的

这份文档用于说明 KotobaLink 当前项目的产品定位、用户角色、核心目标和 MVP 范围。

它的作用是为后续前端、后台和后端开发提供统一的产品上下文。

## 一、产品定位

KotobaLink 当前定位为一个面向日语学习服务场景的学习支持平台。

从当前 demo 的实现来看，产品重点不是单纯卖课，而是围绕“课程服务交付”建立一套学习陪伴流程，包括：
- 课程浏览与购课
- 课表查看
- lesson 内容查看
- assessment 查看
- 每日学习任务
- 作业提交流程
- 消息通知
- 活动报名
- 学习成长展示

## 二、目标用户

### 1. 学员用户

当前 demo 主要服务的就是学员端用户。

他们的核心需求包括：
- 了解自己当前学习状态
- 查看课程与课表
- 完成日常学习任务
- 提交作业与查看反馈
- 获取活动、课程、任务提醒
- 查看成长轨迹和阶段成果

### 2. 运营 / 教务 / 管理人员

虽然当前仓库还没有后台，但从产品结构看，后续必然需要后台角色支持：
- 课程管理
- 排课管理
- 活动管理
- 消息管理
- 订单管理
- 会员管理
- 学员档案管理

### 3. 教师角色

后续如果做完整产品，教师侧也会是一个独立业务角色，至少会涉及：
- lesson 内容维护
- 作业点评
- 成长反馈
- assessment 结果管理

## 三、当前产品目标

从当前 demo 的实现与结构整理来看，现阶段产品目标可以归纳为：

1. 用一个可演示的前端原型验证学习服务闭环
2. 提前沉淀多端项目结构，避免后续推翻目录
3. 把核心业务流程梳理清楚，为真实开发做准备
4. 先搭好用户端体验，再反推后台与后端设计

## 四、当前 MVP 范围

当前 MVP 更接近“学员学习服务闭环演示版”，核心范围包括：

### 已覆盖范围

- 首页总览
- 课程列表
- 课程详情
- 购课结果页
- 已报名课程详情
- lesson 详情
- assessment 详情
- 周课表
- 每日任务
- 作业页
- 消息中心
- 个人中心
- 会员中心展示
- 活动页

### 当前未真正落地但后续必需的范围

- 登录鉴权
- 真实订单与支付
- 真实消息推送
- 真实 lesson / assessment 数据后台维护
- 学员作业记录与教师点评
- 后台管理系统
- 教师侧功能

## 五、核心产品闭环

当前项目最核心的产品闭环有 5 条：

1. 课程浏览 -> 购买 -> 支付结果 -> 进入我的课程
2. 课表查看 -> lesson 详情 -> 学习内容查看
3. 每日任务 -> 词汇学习 -> 打卡完成
4. 任务/课表 -> 作业页 -> 提交作业
5. 活动页 -> 报名 -> 课表与消息联动

这些闭环是当前产品最值得优先固化为真实业务的部分。

## 六、后续正式项目的产品目标

当项目从 demo 转向正式产品时，目标应升级为：

### 学员端

- 提供真实学习服务入口
- 让课程、课表、任务、作业、成长真正联动
- 支持长期学习过程中的状态沉淀

### 管理端

- 支持课程、lesson、活动、订单、消息的运营管理
- 支持教务排课和服务交付
- 支持学员学习进度追踪

### 后端

- 提供稳定业务模型和权限体系
- 支持多端统一接口
- 为未来后台和用户前台同时服务

## 七、当前产品文档建议顺序

如果后续要继续补产品文档，建议顺序是：

1. 产品需求说明
2. 页面流程图
3. 数据实体设计
4. 接口清单
5. 后台模块说明

## 八、当前结论

KotobaLink 当前已经不是单纯的视觉 demo，而是一个具备明确学习服务结构的产品雏形。

后续正式开发时，不应该把它当成“重做一个项目”，而应该把它当成：
- 已完成结构整理的前端原型
- 已具备核心闭环的产品草案
- 可以直接反推接口和后台的基础版本
