# Product Requirements

## Document Purpose

This document describes the current product positioning, user roles, product goals, and MVP scope of KotobaLink.

## 1. Product Positioning

KotobaLink is positioned as a learning-support platform for Japanese language learning services.

The current demo covers:
- course browsing and purchase
- schedule viewing
- lesson content viewing
- assessment viewing
- daily learning tasks
- homework submission
- notifications
- activity signup
- growth display

## 2. Target Users

### Student Users

This is the primary user role covered by the current demo.

Core needs:
- understand the current learning status
- review courses and schedules
- complete daily learning tasks
- submit homework and view feedback
- receive activity, course, and task reminders
- review growth progress

### Operations / Academic / Admin Roles

These roles are required for the future admin application.
- course management
- schedule management
- activity management
- message management
- order management
- membership management
- student archive management

### Teacher Role

This role is required in a fuller product scope.
- lesson content maintenance
- homework feedback
- growth feedback
- assessment result management
